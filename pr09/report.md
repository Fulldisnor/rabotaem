# ПР №9. Следы вредоносного ПО в Linux

## 1. Что было посажено

| Механизм | Место | Команда/файл |
|----------|-------|-------------|
| Cron | crontab пользователя | /tmp/.hidden_malware/backdoor.sh |
| Systemd | ~/.config/systemd/user/ | system-helper.service |
| Shell profile | ~/.bashrc | /tmp/.hidden_malware/backdoor.sh |
| Процесс | /tmp/.hidden_malware/ | listener.sh на порту 4444 |

## 2. Что нашли — процессы

**Команда:** `ps aux | grep '/tmp'`

**Результат:** root 9762 0.0 0.0 6940 3292 pts/1 S 22:12 0:00 /bin/bash /tmp/.hidden_malware/listener.sh
**Что подозрительно:** 
- Процесс запущен из скрытой папки `/tmp/.hidden_malware/` (точка в начале имени — признак скрытия)
- `/tmp` — временная директория, легитимные системные сервисы оттуда не запускаются
- Имя файла `listener.sh` указывает на прослушивание соединений (признак backdoor)
- Процесс работает от root, что дает злоумышленнику полные права

## 3. Что нашли — сетевые соединения

**Команда:** `ss -tulnp`

**Подозрительный порт:** 4444

**Процесс:** nc (netcat), PID 9763

**Команда:** `sudo lsof -i :4444`

**Результат:** COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME
nc 9763 root 3u IPv4 8740 0t0 TCP *:4444 (LISTEN)
**Как lsof связывает порт с процессом:** 
Утилита `lsof` (list open files) показывает все открытые файлы и сетевые дескрипторы процессов. При указании `-i :4444` она фильтрует вывод и показывает только процессы, которые держат открытым сетевой сокет на порту 4444. В выводе видно:
- COMMAND (nc) — имя исполняемого файла
- PID (9763) — идентификатор процесса
- FD (3u) — файловый дескриптор 3, открытый на запись (u)
- TYPE/DEVICE — тип соединения IPv4
- NAME — порт 4444 в состоянии LISTEN

Это позволяет напрямую связать сетевой порт с конкретным процессом.

## 4. Что нашли — автозапуск

### Cron
**Вывод crontab -l:** @reboot /tmp/.hidden_malware/backdoor.sh &
*/5 * * * * /tmp/.hidden_malware/backdoor.sh &

**Что подозрительно:** 
- Запуск скриптов из `/tmp` — нетипично для легитимных задач
- `@reboot` — автоматический запуск при загрузке системы (обеспечение персистентности)
- `*/5 * * * *` — запуск каждые 5 минут (механизм выживания: если процесс убьют, он перезапустится)
- Скрытая папка `.hidden_malware` в пути
- Фоновый запуск через `&` чтобы не блокировать cron

### Systemd
**Вывод systemctl --user list-unit-files:** system-helper.service enabled

Какие меры ФСТЭК №17 реализует эта проверка:
АНЗ.2 (Анализ защищенности): Регулярный поиск несанкционированного ПО через анализ процессов (ps aux), проверку автозапуска (cron, systemd, bashrc), мониторинг сетевых соединений (ss -tulnp). Выявление аномальной активности — процессы из /tmp, нестандартные порты.

Проверка открытых файлов процессов через lsof, анализ сетевых соединений, контроль целостности конфигурационных файлов (crontab, bashrc, systemd unit-файлы). Обнаружение удаленных файлов (deleted) через lsof | grep deleted.
ЗИС.17 (Защита информационных систем): Обнаружение и удаление вредоносного кода, контроль целостности системных настроек, мониторинг сетевой активности на предмет неавторизованных соединений (порт 4444). Проверка персистентных механизмов для выявления закреплений в системе.
Выводы
В ходе практической работы были изучены и продемонстрированы основные техники, которые используют вредоносные программы для закрепления в системе Linux:
Множественные механизмы персистентности: Вредонос был прописан одновременно в cron, systemd и ~/.bashrc. Это обеспечивает выживание — если удалить один механизм, другие восстановят вредонос.
Cron — классический и самый простой способ автозапуска. Записи @reboot и */5 * * * * обеспечивают запуск при загрузке и периодический перезапуск. Обнаруживается через crontab -l.
Systemd user-сервисы — современный и менее очевидный механизм. Требует отдельной проверки через systemctl --user, так как не отображается в общем списке системных сервисов. Опция Restart=always обеспечивает автоматический перезапуск.
Shell-профили (~/.bashrc) — запуск при каждом открытии терминала. Часто маскируется под легитимные комментарии ("system update helper"). Сложно обнаружить в большом файле без grep.
Скрытые папки в /tmp — временная директория с папкой .hidden_malware (точка скрывает от ls без флага -a). /tmp часто используется вредоносами так как:
Доступна для записи любому пользователю
Не требует root-прав
Часто исключается из мониторинга
Сетевые backdoor-порты — порт 4444 (классический port для reverse shell). Обнаруживается через ss -tulnp и lsof -i :4444.
